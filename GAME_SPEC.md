# 바다거북 스프 게임 설계 문서

## 게임 개요
매일 바뀌는 단어를 스무고개 방식으로 맞추는 한국어 워들형 게임.
유저는 예/아니오로 답할 수 있는 질문을 해서 단어를 유추한다.

---

## 질문 횟수 규칙

| 구분 | 횟수 |
|---|---|
| 기본 질문 횟수 | 20회 |
| 광고 시청 시 추가 | +10회 |
| 광고 최대 횟수 | 미정 |

- 질문 횟수 소진 시 게임 종료 (패배)
- 광고 버튼 UI는 미리 구현, 현재는 버튼 클릭 후 5초 대기 → +10회 지급
- 실제 광고 연동은 추후 구현 예정

---

## 입력 분류

```
사용자 입력
    │
    ├── 질문 ("빨간색이야?", "동물이야?")
    │       → Gemini 답변 생성 + 유사도 계산
    │       → 질문 횟수 차감
    │
    └── 단어 추측 ("수박")
            → 유사도 계산만 (Gemini 호출 없음)
            → 질문 횟수 차감 없음
```

**질문 판별 기준**
- `?`로 끝나거나 "이야", "인가", "할까", "야" 등 의문형 어미

---

## 정답 조건
- 유사도 점수 100점 (정확히 일치하거나 직접 동의어)

---

## 랭킹
- 오늘 날짜 기준 정답자 순위
- 질문 횟수 적을수록, 시간 빠를수록 상위
- 닉네임 입력 가능 (미입력 시 "익명")

---

## 아키텍처

```
turtlesoup (Next.js, Vercel)
    ├── 게임 UI
    ├── /api/ask       → Gemini 답변 생성
    ├── /api/solve     → 정답 저장 + 랭킹
    ├── /api/ranking   → 랭킹 조회
    └── /api/cron      → 매일 단어 자동 생성 (KST 00:00)

similarity-server (Python FastAPI, Railway)
    └── /similarity    → FastText 유사도 계산

Supabase (PostgreSQL)
    ├── words          → 날짜별 단어
    └── solves         → 정답 기록 + 랭킹

Gemini 2.5 Flash-Lite
    └── 질문 답변 생성 (세션당 20회 한도)
```
